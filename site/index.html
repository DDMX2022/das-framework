⬚

Isolation, proven

Experts share nothing trainable. Grow or delete one, and every other expert is re-proven byte-identical by SHA-256 fingerprint — on every change, recorded in the audit entry.

⟁

Growth, earned

New capacity trains as a candidate in quarantine and ships only if it clears an accuracy floor and a minimum improvement. Saturated experts stay small — the parsimony gate refuses capacity that isn't needed.

⛁

History, verifiable

HMAC- or Ed25519-signed audit chain, bound to the weights across restarts, rollback-detected via a freshness anchor — and exported as a bundle an auditor verifies offline, without trusting you.

How it works

One spec. One command. A governed fleet.

Declare tenants, experts, users, and an escalation policy. das deploy stands it up with every guarantee attached — locally, in Docker, or on Kubernetes.

# client.yaml client: acme backend: lora-minilm tenants: - name: legalco experts: [{name: legal}] - name: medico experts: [{name: medical}] users: - {name: care-agent, role: operator, tenant: medico} escalation: {frontier_llm: claude-sonnet-5, confidence_threshold: 0.7}
$ das deploy client.yaml --save state/ $ python >>> dep.route("care-agent", ... "MRI scan blocked pending urgent review") {expert: 'medical', decision: 'local'} >>> dep.grow("root", "medico", "insurance", ... stage="seed") # others proven untouched >>> dep.germinate("root", eid=2) # promote only if earned >>> dep.offboard("legalco") # provable deletion >>> dep.export_bundle("acme_audit.json")

Queries the fleet can answer stay local on small, isolated experts; low-confidence queries escalate to your frontier LLM. The routing decision is the cost-deflection signal — measured per query, tallied per fleet.
