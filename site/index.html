Real output. Every expert is a LoRA adapter on a frozen transformer's own attention layers — growing one is real fine-tuning, and the others don't move a byte.

Isolation, proven

Experts share nothing trainable. Grow or delete one, and every other expert is re-proven byte-identical by SHA-256 fingerprint — on every change, recorded in the audit entry. Certification survives change.

Growth, earned

New capacity trains as a candidate in quarantine and ships only if it clears an accuracy floor and a real improvement. Saturated experts stay small — the parsimony gate refuses capacity that isn't needed.

History, verifiable

HMAC- or Ed25519-signed audit chain, bound to the weights across restarts, rollback-detected via a freshness anchor — exported as a bundle an auditor verifies offline, without trusting you.

The whole plane

Everything around the adapters — so you don't build it yourself

Isolated adapters are table stakes; teams re-invent the surrounding plane at every company. DAS ships it as one governed system.

Task-free routing

Queries embed once and route to the right specialist with a confidence score — no task labels at inference, provenance on every answer.

Multi-tenancy + RBAC

Tenant-scoped operators, auditors, viewers. A careplus operator cannot touch a fintrust expert — denials are logged like everything else.

Provable deletion

Right-to-be-forgotten at expert or whole-tenant granularity — survivors re-proven byte-identical, the removal signed into the chain.

Escalation + cost deflection

Low-confidence queries escalate to your frontier LLM; everything else stays local on small experts. The decision is a measurable per-query cost signal.

Rank-ladder germination

Experts start at 770 parameters and earn LoRA rank through accuracy gates. Measured: most experts never need to grow at all.

Teacher-driven learning

An LLM (Ollama / OpenAI-compatible) writes the lesson corpus; the adapter trains on it in quarantine; policy decides if it ships.

Restart-safe persistence

Byte-exact save/load bound to the signed log; a rolled-back snapshot is detected and refused at boot.

Licensing + consoles

Ed25519-signed entitlements enforced for the fleet's lifetime; a Rancher-style multi-fleet console with session login; growth dashboards.

How it works

One spec. One command. A governed fleet.

Declare tenants, experts, users, and an escalation policy. das deploy stands it up with every guarantee attached — locally, in Docker, or on Kubernetes behind the reference TLS/OIDC gateway.

# client.yaml client: acme backend: lora-minilm tenants: - name: legalco experts: [{name: legal}] - name: medico experts: [{name: medical}] users: - {name: care-agent, role: operator, tenant: medico} escalation: {frontier_llm: claude-sonnet-5, confidence_threshold: 0.7}
$ das deploy client.yaml --save state/ >>> dep.route("care-agent", ... "MRI scan blocked pending urgent review") {expert: 'medical', decision: 'local'} >>> dep.grow("root", "medico", "insurance", ... stage="seed") # others proven untouched >>> dep.germinate("root", eid=2) # promote only if earned >>> dep.offboard("legalco") # provable deletion >>> dep.export_bundle("acme_audit.json")
