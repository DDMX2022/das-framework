Using DAS with a real LLM

In production, the tiny demo experts become real LLM specialists: LoRA adapters, RAG indexes, fine-tuned heads, or separate model endpoints.

production wiring

1. Agent builds the task

Your LangGraph or bootcamp app gathers the user message, tools, policy, and tenant token. It does not decide which private model weights are safe to use.

2. DAS selects the specialist

DAS verifies the tenant, routes the request, and opens only one specialist. That specialist can wrap a real LLM call like claim_llm.invoke().

3. Answer returns with proof

The app gets the answer plus provenance: which tenant, which specialist, confidence, and audit evidence. Other specialists never saw the prompt.

state = graph.invoke({"message": user_text, "tenant": "careplus"})

# inside the DAS node:
expert = das.route(tenant=state["tenant"], embedding=state["embedding"])
answer = expert.real_llm.invoke(state["message"])  # only this specialist runs
return {"answer": answer, "served_by": expert.name, "tenant": expert.tenant}
